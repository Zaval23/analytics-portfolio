# UC-01 — Создание квеста

## Общие сведения

| Поле | Значение |
|------|----------|
| **Цель** | Автор создаёт новый квест и заполняет базовые данные и граф шагов |
| **Акторы** | Автор *(Администратор — при необходимости)* |
| **Предусловия** | Пользователь аутентифицирован с ролью «Автор» |
| **Постусловия** | Квест сохранён в статусе «Черновик» или отправлен на публикацию |

## Основной сценарий

1. Автор открывает раздел «Мои квесты».
2. Система отображает список существующих квестов и кнопку «Создать».
3. Автор нажимает «Создать».
4. Система создаёт запись квеста и открывает редактор.
5. Автор вводит название и описание.
6. Автор добавляет шаги и переходы *(в соответствии с редактором графа)*.
7. Автор сохраняет черновик.
8. Система сохраняет данные и подтверждает успех.

## Альтернативные потоки

**A1. Ошибка валидации при сохранении**

- На шаге 7 система обнаруживает нарушение бизнес-правил графа.
- Система показывает сообщение; квест остаётся в черновике; автор исправляет граф.

**A2. Отмена**

- Автор закрывает редактор без сохранения — система запрашивает подтверждение потери данных *(если политика такая)*.
