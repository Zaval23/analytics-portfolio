# UC-03 — Валидация графа квеста

## Общие сведения

| Поле | Значение |
|------|----------|
| **Цель** | Убедиться, что граф квеста допускает корректное прохождение и соответствует бизнес-правилам до публикации |
| **Акторы** | Система *(инициирует Автор при «Опубликовать» или автоматически при сохранении)* |
| **Предусловия** | Для квеста заданы узлы и рёбра в БД |
| **Постусловия** | Результат проверки сохранён / показан автору; публикация разрешена или запрещена |

## Бизнес-правила проверки (примерный набор — уточните)

- Из стартового узла существует путь ко всем обязательным узлам *(или иная политика достижимости)*.
- Нет нежелательных циклов *(или циклы разрешены — уточните)*.
- Каждый нестартовый узел имеет хотя бы одно входящее ребро.
- Нет «висячих» узлов вне основного сценария *(по политике продукта)*.

## Основной сценарий

1. Автор инициирует публикацию или система запускает проверку после сохранения графа.
2. Система загружает структуру графа *(узлы, рёбра)*.
3. Система выполняет алгоритмы проверки *(в т.ч. рекурсивный обход / CTE — см. `sql_queries/validation_cte.sql`)*.
4. Если все проверки успешны — квест может быть опубликован.
5. Если есть ошибки — система возвращает список проблем автору.

## Альтернативные потоки

**A1. Обнаружен цикл при запрещённых циклах**

- На шаге 3 находится цикл; публикация блокируется; сообщение «Обнаружен цикл …».

**A2. Недостижимый узел**

- Узел не достижим из стартового — блокировка с указанием идентификатора узла.
