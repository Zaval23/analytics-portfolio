# UC-03: Валидация графа

| **Поле** | **Значение** |
|----------|---------------|
| **Название** | Валидация графа квеста |
| **Актор** | Система (по запросу автора) |
| **Предусловие** | Квест содержит хотя бы одну вершину и стартовую вершину |
| **Триггер** | Автор нажимает кнопку "Проверить" или "Опубликовать" |
| **Успешный постусловие** | Все вершины достижимы от стартовой |

## Основной сценарий

1. Автор запрашивает валидацию (нажатие кнопки)
2. Система определяет стартовую вершину квеста
3. Система запускает алгоритм обхода графа (BFS/DFS) от стартовой вершины:
   - Перебирает все исходящие ребра
   - Посещает все достижимые вершины
   - Защита от циклов: не посещать одну вершину дважды
4. Система собирает множество всех достигнутых вершин
5. Система сравнивает с множеством всех вершин квеста
6. Если все вершины достижимы → возвращает результат "Валидация пройдена"
7. Если есть недостижимые вершины → возвращает список проблемных вершин

## Альтернативные потоки

**7a. Обнаружены недостижимые вершины**
1. Система формирует список вершин, в которые нельзя попасть
2. Для каждой вершины указывает ID и текст (для удобства автора)
3. Система блокирует публикацию (если валидация вызывалась перед публикацией)
4. Автор исправляет граф (добавляет ребра) и повторяет валидацию

**5a. В графе есть циклы, но все вершины достижимы**
1. Циклы допустимы (игрок может возвращаться)
2. Система предупреждает о циклах, но не блокирует публикацию

## Дополнительные проверки

| Проверка | Действие при ошибке |
|----------|---------------------|
| Наличие хотя бы одной хорошей концовки | Предупреждение (не блокирует публикацию) |
| Все условия ссылаются на существующие предметы/счетчики | Ошибка, блокировка публикации |
| Значения счетчиков не выходят за min/max | Подрезать автоматически |

## Исключения

| Исключение | Действие системы |
|------------|------------------|
| Квест не содержит ни одной вершины | Ошибка "Невозможно валидировать пустой квест" |
| Не указана стартовая вершина | Ошибка "Не задана стартовая вершина" |
| Глубина рекурсии превысила 1000 | Предположить наличие цикла, выдать предупреждение |

## Бизнес-правила

- Квест не может быть опубликован, если есть недостижимые вершины
- Валидация должна выполняться за < 1 секунды для графа из 100 вершин
- Алгоритм должен защищать от бесконечных циклов (максимальная глубина 50)
