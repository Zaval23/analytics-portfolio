# UC-02 — Прохождение квеста

## Общие сведения

| Поле | Значение |
|------|----------|
| **Цель** | Игрок проходит опубликованный квест от стартового шага к завершению |
| **Акторы** | Игрок |
| **Предусловия** | Квест опубликован; игрок авторизован *(или допускается гость — уточните)* |
| **Постусловия** | Прогресс сохранён; история переходов доступна для продолжения |

## Основной сценарий

1. Игрок открывает страницу квеста из каталога.
2. Система показывает описание и кнопку «Начать» / продолжает с сохранённого шага.
3. Игрок начинает или продолжает прохождение.
4. Система отображает текущий шаг и доступные варианты переходов.
5. Игрок выбирает действие / переход.
6. Система фиксирует выбор, переводит на следующий шаг *(или завершает квест)*.
7. Шаги 4–6 повторяются до терминального шага.
8. Система показывает экран завершения *(награда, статистика — по продукту)*.

## Альтернативные потоки

**A1. Нет доступных переходов**

- На шаге 4 система не находит исходящих рёбер — отображается ошибка конфигурации квеста / конец с ошибкой.

**A2. Прерывание сессии**

- Игрок закрывает клиент; при возврате система восстанавливает последний шаг из сессии прохождения.
