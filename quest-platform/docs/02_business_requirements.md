# Business Requirements Document (BRD) - Бизнес-требования | Платформа квестов

## BR-1: Создание контента

| **Поле** | **Значение** |
|----------|---------------|
| **Название** | Создание квестов авторами |
| **Описание** | Автор должен иметь возможность создавать квесты с нелинейным сюжетом, где игрок может выбирать действия и получать разные концовки |
| **Приоритет** | **High** (без этого нет смысла в платформе) |
| **Бизнес-ценность** | Привлечение авторов → создание контента → привлечение игроков |
| **Успешный критерий** | Автор создает квест из 10 вершин и 15 ребер за **15 минут** |
| **Зависимости** | Никаких |

---

## BR-2: Вовлеченность игроков

| **Поле** | **Значение** |
|----------|---------------|
| **Название** | Прохождение квестов до конца |
| **Описание** | Игроки должны иметь возможность проходить квесты и достигать хороших концовок |
| **Приоритет** | **High** |
| **Бизнес-ценность** | Удовлетворенные игроки возвращаются на платформу |
| **Успешный критерий** | **70%** начатых квестов заканчиваются хорошей концовкой |
| **Зависимости** | BR-1 (нужны квесты) |

---

## BR-3: Сохранение прогресса

| **Поле** | **Значение** |
|----------|---------------|
| **Название** | Возможность продолжить прохождение позже |
| **Описание** | Авторизованный игрок может прервать прохождение квеста и продолжить с того же места на любом устройстве |
| **Приоритет** | **Medium** |
| **Бизнес-ценность** | Удобство для игроков → повторные визиты |
| **Успешный критерий** | **90%** сессий восстанавливаются корректно |
| **Зависимости** | Система авторизации |

---

## BR-4: Качество контента

| **Поле** | **Значение** |
|----------|---------------|
| **Название** | Валидация квестов перед публикацией |
| **Описание** | Система должна проверять, что все сцены (вершины) достижимы от начала квеста, и не дать опубликовать "сломанный" квест |
| **Приоритет** | **High** |
| **Бизнес-ценность** | Игроки не попадают в тупики → позитивный опыт |
| **Успешный критерий** | **100%** опубликованных квестов проходимы от начала до концовки |
| **Зависимости** | BR-1 |

---

## BR-5: Каталог и поиск

| **Поле** | **Значение** |
|----------|---------------|
| **Название** | Поиск книг и квестов |
| **Описание** | Пользователь (даже без регистрации) может искать книги по названию, жанру или автору |
| **Приоритет** | **Medium** |
| **Бизнес-ценность** | Новые пользователи находят контент без барьера входа |
| **Успешный критерий** | Поиск возвращает результаты за **< 1 секунды** |
| **Зависимости** | Никаких |

---

## BR-6: Обратная связь от игроков

| **Поле** | **Значение** |
|----------|---------------|
| **Название** | Оценка книг и комментарии |
| **Описание** | Авторизованный игрок может оценить книгу (1-5 звезд) и оставить текстовый комментарий |
| **Приоритет** | **Low** (можно в MVP, можно позже) |
| **Бизнес-ценность** | Сообщество, обратная связь для авторов |
| **Успешный критерий** | **30%** прошедших квест оставляют оценку |
| **Зависимости** | BR-1, система авторизации |

---

## Приоритеты (сводка)

| Приоритет | Требования |
|-----------|------------|
| **High** | BR-1 (создание квестов), BR-2 (прохождение), BR-4 (валидация) |
| **Medium** | BR-3 (сохранение прогресса), BR-5 (поиск) |
| **Low** | BR-6 (оценки и комментарии) |
