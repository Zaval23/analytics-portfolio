# Vision & Scope - Концепция и границы проекта | Платформа для создания квестов

## 1. Бизнес-цели
- Создать веб-платформу для интерактивных квестов в формате "выбери своё приключение"
- Дать авторам инструмент для создания нелинейных сюжетов с предметами и скрытыми счетчиками
- Дать игрокам возможность проходить квесты (анонимно или с сохранением прогресса)
- Создать сообщество вокруг пользовательского контента (оценки, комментарии)

## 2. Целевая аудитория
| Роль | Описание | Потребности |
|------|----------|-------------|
| **Авторы** | Создатели контента | Гибкий редактор графов, понятные условия, валидация |
| **Игроки (анонимные)** | Проходят без регистрации | Быстрый старт, понятный интерфейс |
| **Игроки (авторизованные)** | Зарегистрированные пользователи | Сохранение прогресса, оценки, комментарии |

## 3. Что входит в проект (In-Scope) — MVP

### Функциональность для авторов
- Создание квестов в виде ориентированных графов (вершины + ребра)
- Система предметов (видимых) и счетчиков (скрытых)
- Условия на ребрах (проверка предметов/счетчиков)
- Эффекты в вершинах (выдача предметов, изменение счетчиков)
- Типы вершин: обычная, с таймером, концовка (хорошая/плохая)
- Лимит переходов по ребрам
- Валидация графа (проверка достижимости всех вершин от стартовой)
- Объединение квестов в книги
- Связывание хороших концовок с началом других квестов

### Функциональность для игроков
- Поиск книг по названию, жанру, автору
- Прохождение квестов (выбор действий, получение предметов)
- Автосохранение прогресса для авторизованных
- Оценка книг (1-5) и комментарии

## 4. Что не входит в проект (Out-of-Scope)
- Мобильное приложение (только веб-версия)
- Многопользовательские квесты (кооператив)
- Чат между игроками
- Система рекомендаций на основе ML
- Монетизация / платежная система
- Экспорт/импорт квестов в сторонние форматы
- Редактор графов с drag-and-drop (для MVP — текстовый редактор)
- Система достижений и ачивок

## 5. Ключевые метрики успеха (KPI)
| Метрика | Целевое значение |
|---------|------------------|
| Время создания квеста (10 вершин) | < 15 минут |
| Процент успешных валидаций графа | > 95% |
| Конверсия в хорошую концовку | > 70% |
| Время ответа API | < 500 мс |
| Восстановление сессии после разрыва | 100% |

## 6. Ограничения
- Бюджет: учебный проект, без финансирования
- Сроки: 3 месяца на MVP
- Технологии: PostgreSQL, Python/FastAPI, React

## 7. Успех проекта
Проект считается успешным, если:
1. Автор может создать и опубликовать квест без ошибок
2. Игрок может пройти квест от начала до хорошей концовки
3. Прогресс авторизованного игрока сохраняется между сессиями
