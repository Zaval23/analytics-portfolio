
## 1. Общее описание

Реализуется платформа для создания и прохождения квестов в формате ориентированных графов. Авторы могут проектировать нелинейные сюжеты с системой инвентаря, скрытыми переменными (счетчиками), временными ограничениями и ветвлением сюжета. Игроки могут проходить квесты как анонимно, так и с сохранением прогресса после авторизации, а также оценивать и комментировать книги.


# Краткий глоссарий 
| Термин                   | Определение                                                          |
| ------------------------ | -------------------------------------------------------------------- |
| **Автор**                | Пользователь, создающий книги и квесты, настраивающий логику графа   |
| **Анонимный игрок**      | Игрок без авторизации, прогресс не сохраняется между сессиями        |
| **Авторизованный игрок** | Игрок с сохранением прогресса, может оценивать и комментировать      |
| **Книга**                | Коллекция квестов с метаданными (название, описание, жанры, обложка) |
| **Квест**                | Отдельный эпизод внутри книги в виде графа из вершин и ребер         |
| **Граф квеста**          | Ориентированная структура из вершин и ребер, описывающая сюжет       |
| **Вершина**              | Узел графа с сюжетным текстом (обычная, концовка, с таймером)        |
| **Ребро**                | Переход между вершинами с текстом действия и условиями               |
| **Концовка**             | Вершина, завершающая квест (хорошая или плохая)                      |
| **Хорошая концовка**     | Успешное завершение, открывает доступ к следующим квестам            |
| **Плохая концовка**      | Неуспешное завершение, прогресс не засчитывается                     |
| **Вершина с таймером**   | Вершина с ограничением времени на выбор действия                     |
| **Инвентарь**            | Набор предметов игрока в текущей сессии                              |
| **Предмет**              | Видимый объект, который можно получить, использовать, потерять       |
| **Счетчик**              | Скрытый числовой параметр с мин/макс значениями                      |
| **Условие перехода**     | Проверка предметов или счетчиков для доступа к ребру                 |
| **Cобытия в вершине**    | Изменение предметов/счетчиков при входе в вершину                    |
| **Лимит переходов**      | Максимальное число использований ребра за сессию                     |
| **Видимо всегда**        | Ребро показывается всегда (даже если недоступно)                     |
| **Условно видимо**       | Ребро показывается только когда доступно                             |
| **Сессия**               | Текущее прохождение квеста (позиция, инвентарь, счетчики)            |
| **Связь квестов**        | Переход из хорошей концовки в начало другого квеста                  |
| **Скрытый квест**        | Квест, доступный только через связь (не виден в каталоге)            |
| **Валидация графа**      | Проверка достижимости всех вершин от стартовой                       |


### Основные функции

- Создание квестов в виде графов (вершины + ребра)

- Управление видимыми предметами и скрытыми счетчиками

- Настройка условий прохождения на ребрах (проверка предметов/счетчиков)

- Настройка эффектов(событий) в вершинах (выдача предметов, изменение счетчиков)

- Система таймеров на принятие решения

- Лимиты на количество переходов по вершинам/ребрам

- Объединение квестов в книги (сериализация)

- Связывание хороших концовок с новыми квестами (перенос прогресса)

- Валидация графов (проверка достижимости всех вершин)

- Прохождение квестов с сохранением состояния

- Поиск книг по названию, жанру, автору

- Оценка и комментирование книг



## 2. Роли пользователей

### 2.1 Автор квестов (Author)

**Описание:** Пользователь, который создает контент - книги и квесты.

**Возможности:**

- Создавать, редактировать и удалять свои книги

- Создавать, редактировать и удалять свои квесты

- Работать с графом: создавать вершины и соединять их ребрами

- Создавать предметы и счетчики для квестов

- Настраивать условия на ребрах (проверка предметов/счетчиков)

- Настраивать эффекты в вершинах (выдача предметов, изменение счетчиков)

- Задавать типы вершин (концовка с типом good/bad, вершина с таймером)

- Устанавливать лимиты на количество переходов по ребрам

- Объединять квесты в книги

- Настраивать метаданные книги (название, описание, жанры, обложка)

- Устанавливать видимость квестов (открытый/скрытый)

- Связывать хорошие концовки с началом других квестов

- Запускать валидацию графа перед публикацией

- Получать подсказки при создании условий (только существующие предметы/счетчики)

- Просматривать статистику прохождений своих квестов (опционально)


### 2.2 Игрок (анонимный)

**Описание:** Пользователь, который проходит квесты без авторизации.

**Возможности:**

- Искать книги в каталоге по фильтрам (название, жанр, автор)

- Просматривать информацию о книгах (описание, жанры, обложка, список квестов)

- Начинать прохождение квеста

- Проходить квест (читать текст вершин, выбирать действия)

- Видеть свой инвентарь (предметы)

- Получать предметы и изменять счетчики по ходу игры

- Завершать квест (хорошая или плохая концовка)


**Ограничения:**

- Прогресс не сохраняется между сессиями (живет пока открыт браузер / пока жива cookie-сессия)

- Не может оценивать книги

- Не может оставлять комментарии

- Не может сохранять прогресс



### 2.3 Игрок (авторизованный)

**Описание:** Зарегистрированный пользователь, проходящий квесты.

**Возможности:**

- Все возможности анонимного игрока

- Сохранять прогресс прохождения (сессия привязана к аккаунту)

- Продолжать прохождение с места остановки на любом устройстве

- Оценивать книги (лайк/рейтинг по шкале, например 1-5)

- Оставлять текстовые комментарии к книгам (плоская структура, без вложенных ответов)

- Просматривать историю пройденных квестов

- Видеть свои оценки и комментарии в профиле


**Наследование прав:** Авторизованный игрок наследует все права анонимного игрока.


## 3. Функциональные требования

### 3.1. Модуль управления квестами (Редактор графов)

#### 3.1.1. Базовые элементы графа

| Требование                     | Описание                                                                                                                                                                                                                                                                                                   |
| ------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Создание вершин                | Автор должен иметь возможность создавать вершины графа. Каждая вершина имеет уникальный идентификатор в рамках квеста и текстовое описание.                                                                                                                                                                |
| Создание ребер                 | Автор должен иметь возможность соединять вершины направленными ребрами (переходами). Каждое ребро содержит текст действия (что видит игрок) и ссылки на "from" и "to" вершины.                                                                                                                             |
| Типы вершин                    | Автор должен иметь возможность задавать тип вершины: **обычная, обычная с таймером, концовка**.                                                                                                                                                                                                            |
| Типы концовок                  | Если вершина помечена как концовка, автор должен указать её тип: **хорошая** (успешное завершение, открывает доступ к следующим квестам, что также позволяет автору указывать в следующих частях это концовку, как связь со стартовой) или **плохая** (прерывает прохождение, не засчитывается как успех). |
| Множественные хорошие концовки | Система должна поддерживать наличие нескольких хороших концовок в рамках одного квеста.                                                                                                                                                                                                                    |

#### 3.1.2. Инвентарь и счетчики (Переменные)

| Требование          | Описание                                                                                                                                                                                                           |
| ------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Типы ресурсов       | Автор должен иметь возможность создавать два типа ресурсов: **видимые предметы** (отображаются в инвентаре игрока) и **скрытые счетчики** (не отображаются игроку, используются для логики, например «репутация»). |
| Границы счетчиков   | Для каждого счетчика автор должен иметь возможность задать **минимальное и максимальное** допустимое значение. Значение счетчика не может выйти за эти границы.                                                    |
| Выдача предметов    | Автор должен иметь возможность настроить выдачу предметов игроку при входе в вершину (указать список предметов и их количество).                                                                                   |
| Изменение счетчиков | Автор должен иметь возможность изменять значения счетчиков при посещении вершины. Поддерживаемые операции: **сложение** и **вычитание** заданного числа.                                                           |

#### 3.1.3. Условия и видимость переходов (Ребра)

| Требование                   | Описание                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| ---------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Условия на ребрах и вершинах | Автор должен иметь возможность задавать **условия** для перехода по ребру. Условие может как просто проверять, так и изменять значение, причём может изменить либо на заданное значение, либо на всё допустимое значение (операторы >=, <=, =, >, <) или значения скрытых счетчиков (те же операторы). В **вершинах** может происходить **ТОЛЬКО** выдача предметов,  не может быть условий, при которых нужно забирать предметы в вершине, т.к. для перехода в определенную вершину мы совершаем переход по ребру, где взимается плата( если бы у нас условно не хватало ресурсов, чтобы перейти, то мы бы просто не перешли) |
| Лимит переходов              | Автор должен иметь возможность установить **лимит на количество переходов** по ребру (например, только 1 раз за всю игру). После достижения лимита ребро становится недоступным.                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| Видимость ребер              | Автор должен иметь возможность настроить **видимость** ребра в интерфейсе игрока: **видимо всегда** (текст перехода показывается всегда, даже если условия не выполнены, но переход недоступен) или **видимо только при выполнении условий** (если условий нет или они соблюдены - ребро видно, если нет - скрыто).                                                                                                                                                                                                                                                                                                            |

#### 3.1.4. Специальные механики вершин

| Требование      | Описание                                                                                                                                                                                                                                                          |
| --------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Таймер на выбор | Автор должен иметь возможность задать для вершины параметр **«Время на выбор»** (в секундах). Если игрок не совершает выбор за отведенное время, система автоматически выбирает **случайный доступный переход** из тех, что доступны на момент истечения таймера. |



### 3.2. Модуль управления книгами (Сериализация квестов)

| **Требование**                                | **Описание**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| --------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Объединение в книги**                       | Автор должен иметь возможность объединять несколько квестов в «Книгу». Книга является основной единицей каталога.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| **Метаданные книги**                          | Автор должен иметь возможность настраивать метаданные книги: название, описание, жанры, обложка.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| **Видимость квестов**                         | Автор должен иметь возможность устанавливать видимость квеста в рамках книги: **Открытый** (доступен для прохождения сразу) или **Скрытый** (становится доступен только после выполнения определенных условий - связи с концовкой другого квеста).                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| **Связь квеста с концовкой**                  | Автор должен иметь возможность связать начало скрытого квеста Б с хорошей концовкой квеста А. При этом:                                                • Указывается, из какой хорошей концовки квеста А открывается доступ к квесту Б                                                                                           • Указывается стартовая вершина в квесте Б                                              • При достижении игроком указанной концовки квеста А - квест Б становится доступен для прохождения                                                      • Инвентарь и счетчики из квеста А **переносятся** в квест Б (если предметы/счетчики не противоречат условиям видимости) |
| **Перенос предметов без связывания концовок** | Автор должен иметь возможность настроить **простой перенос предметов и счетчиков** между квестами без привязки к концовке. В этом случае:                                                                                                   • Указывается квест-источник А и квест-приемник Б                                • Определяется список предметов и счетчиков, которые переносятся   • Перенос происходит автоматически при начале квеста Б (если игрок уже проходил квест А)                                                                                                                                                                                                                  |
| **Условия видимости скрытого квеста**         | Для скрытого квеста автор должен иметь возможность задать **условия видимости**, определяющие, когда квест становится доступным. Условия могут проверять:                                                                             • Наличие определенных предметов                                                          • Значения счетчиков из связанных квестов                                               • Прохождение конкретных концовок других квестов                                                                                                                                                                                                                                  |
| **Проверка достижимости условий**             | Система должна проверять возможность достижения заданных условий видимости:                                                                                      • Анализировать граф квеста-источника                                                    • Проверять, можно ли достичь указанной концовки с накоплением требуемых предметов/счетчиков                                                                • При невозможности достижения условий - выдавать предупреждение автору                                                                                                                                                                                                             |
| **Отображение переносимых предметов**         | В интерфейсе настройки квеста система должна отображать, какие предметы и счетчики будут перенесены из других квестов, а также стартовые значения инвентаря (с учетом переносов).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| **Обновление стартового инвентаря**           | Стартовый инвентарь квеста формируется динамически:                        • Базовые значения (заданные автором для квеста)                                 • Плюс переносимые предметы и счетчики из связанных квестов (если игрок их прошел) Важно: Если переносимое значение счетчика не соответствует ограничениям (min/max) квеста-приемника, оно автоматически подрезается под допустимые границы.                                                                                                                                                                                                                                                                                                  |

## 4. Бизнес-процессы

### 4.1 Создание квеста (Автор)

**Роль:** Автор  
**Предусловия:** Автор авторизован, имеет роль "Автор"

**Основной сценарий:**
1. Автор создает новый квест (или открывает существующий черновик)

2. Система создает пустой граф со стартовой вершиной

3. Автор добавляет вершины, наполняет их текстом

4. Автор соединяет вершины ребрами, задает текст действия

5. При необходимости автор создает предметы и счетчики

6. Автор настраивает условия на ребрах (выбирает предметы/счетчики из списка)

7. Автор настраивает видимость ребер

8. Автор настраивает эффекты в вершинах (выдача предметов, изменение счетчиков)

9. Автор помечает вершины как концовки (с указанием типа) или добавляет таймеры

10. Автор сохраняет черновик квеста

11. Система сохраняет текущее состояние как черновик, позволяя автору продолжить редактирование позже

12. При попытке публикации автор запускает валидацию графа

13. Система проверяет:
    - Достижимость всех вершин от стартовой
    - Наличие хотя бы одной хорошей концовки
    - Целостность условий и эффектов (все предметы/счетчики существуют)

14. Если валидация не пройдена — система блокирует публикацию, указывает на ошибки, автор возвращается к редактированию черновика

15. Если валидация пройдена — автор публикует квест
 **nice to have**
16. Система создает новую **версию** квеста:
    - Опубликованная версия становится доступной для игроков
    - Предыдущие версии сохраняются (игроки, начавшие прохождение ранее, продолжают использовать свою версию)
    - Автор может просматривать, но не изменять старые версии

17. Если публикация квеста приводит к изменению книги (добавление нового квеста или изменение существующего), версия книги обновляется

18. Если в книгу добавляется только новый квест (без изменения существующих), версия книги не меняется


### 4.2 Создание книги и связывание квестов (Автор)

**Роль:** Автор  
**Предусловия:** Созданы квесты

**Основной сценарий:**

1. Автор создает новую книгу

2. Заполняет метаданные: название, описание, жанры, обложка

3. Добавляет созданные квесты в книгу

4. Для каждого квеста устанавливает видимость (открытый/скрытый)

5.  Для каждого **скрытого квеста** указывает, какая хорошая концовка какого квеста его открывает:
    - Выбирает квест, в котором находится открывающая концовка
    - Выбирает хорошую концовку этого квеста
    - Выбирает стартовую вершину в скрытом квесте (с чего начнется прохождение)

6. Публикует книгу




### 4.3 Прохождение квеста (Игрок)

**Роль:** Игрок (анонимный или авторизованный)  
**Предусловия:** Книга опубликована, квест доступен

**Основной сценарий:**

1. Игрок находит книгу через поиск или каталог

2. Выбирает квест для прохождения

3. Система создает игровую сессию:
    
    - Для анонима - сессия привязана к браузеру (cookie/device_id)
    
    - Для авторизованного - сессия привязана к user_id
    
4. Система инициализирует инвентарь и счетчики (начальные значения)

5. Игрок попадает в стартовую вершину, видит текст и список доступных действий (ребер)

6. Игрок выбирает действие:

    - Если в вершине есть таймер - запускается обратный отсчет
    
    - Если таймер истек без выбора - система выбирает случайное доступное ребро
    
7. Система проверяет условия на выбранном ребре:
    
    - Если условия не выполнены - действие недоступно (сообщение об ошибке)
    
    - Если выполнены - применяются эффекты целевой вершины (выдача предметов, изменение счетчиков)
    
8. Система обновляет позицию игрока (текущая вершина)

9. Шаги 5-8 повторяются до достижения концовки

10. При достижении концовки:
    
    - Если концовка хорошая и есть связь с другим квестом - игроку предлагается либо сразу перейти в связанный квест, либо вернуться к книге, чтобы он увидел, что новый квест доступен
    
    - Если концовка хорошая без связи - квест завершен успешно
    
    - Если концовка плохая - квест завершен неуспешно
    
11. Для авторизованного игрока прогресс сохраняется автоматически после каждого шага




## nice to have##
4.4 Сохранение и продолжение прогресса (Авторизованный игрок)
**Роль:** Авторизованный игрок  
**Предусловия:** Игрок начал прохождение квеста
**Основной сценарий:**
1. Система автоматически сохраняет состояние сессии после каждого перехода
2. Игрок выходит из приложения
3. При следующем входе игрок видит список активных сессий (незавершенные квесты)
4. Игрок выбирает сессию для продолжения
5. Система восстанавливает состояние:
    - Текущая вершина
    - Инвентарь (предметы с количеством)
    - Состояния счетчиков
    - История посещенных вершин
    - Количество использований ребер с лимитами переходов
    - Оставшееся время (если вершина имеет активный таймер)
6. Игрок продолжает прохождение с того же места
**примечание:**  
Для полного восстановления сессии необходимо сохранять не только текущую вершину и ресурсы игрока, но и историю прохождения (посещенные вершины, счетчики использований ребер, состояние таймеров). Это обеспечивает корректную работу лимитов переходов, логики "нельзя вернуться" и механик с таймером при продолжении игры.


### 4.5 Поиск книг (Все пользователи)

**Роль:** Любой пользователь (анонимный или авторизованный)

**Основной сценарий:**

1. Пользователь вводит поисковый запрос (название) или применяет фильтры (жанр, автор)

2. Система выполняет поиск по каталогу книг

3. Возвращает список книг, соответствующих критериям

4. Для каждой книги отображается: название, обложка, автор, жанры, рейтинг, краткое описание

5. Пользователь может перейти на страницу книги и увидеть список квестов в ней


### 4.6 Оценка и комментирование (Авторизованный игрок)

**Роль:** Авторизованный игрок  
**Предусловия:** Игрок прошел хотя бы один квест из книги

**Основной сценарий (оценка):**

1. На странице книги игрок выбирает оценку (например, от 1 до 5)

2. Система сохраняет оценку, привязывая к игроку и книге

3. Система пересчитывает средний рейтинг книги

4. Новая оценка отображается в общем рейтинге


**Основной сценарий (комментарий):**

1. На странице книги игрок вводит текст комментария

2. Система сохраняет комментарий с указанием автора и даты

3. Комментарий появляется в общем списке под книгой

4. Другие пользователи могут читать комментарии (без возможности ответа)



### 4.7 Валидация графа (Система)

**Роль:** Система (по запросу автора)

**Основной сценарий:**

1. Автор запрашивает валидацию квеста

2. Система получает стартовую вершину (from `starting_vertex_id`)

3. Запускает алгоритм обхода графа (BFS/DFS) от стартовой вершины по всем ребрам

4. Собирает множество достигнутых вершин

5. Сравнивает с множеством всех вершин квеста

6. Если есть недостижимые вершины:
    
    - Формирует список проблемных вершин
    
    - Блокирует сохранение
    
    - Возвращает ошибку с указанием вершин
    
7. Дополнительно проверяет:
    
    - Все ли предметы/счетчики, используемые в условиях и эффектах, существуют
    
    - Есть ли хотя бы одна хорошая концовка (предупреждение, не ошибка)
    
8. Возвращает результат валидации
