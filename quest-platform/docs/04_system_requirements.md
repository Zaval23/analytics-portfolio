# System Requirements - Системные требования| Платформа квестов

## 1. Производительность (Performance)

| Требование | Целевое значение | Способ измерения |
|------------|------------------|------------------|
| Время ответа API (95% запросов) | < 500 мс | APM-мониторинг (New Relic / Prometheus) |
| Валидация графа (100 вершин) | < 1 секунда | EXPLAIN ANALYZE в PostgreSQL |
| Загрузка квеста с графом | < 2 секунд | Chrome DevTools (Network tab) |
| Поддержка одновременных сессий | 1000 | Нагрузочное тестирование (JMeter / k6) |

---

## 2. Надежность (Reliability)

| Требование | Описание |
|------------|----------|
| Автосохранение прогресса | Состояние игрока сохраняется в БД после каждого перехода по ребру |
| Восстановление сессии | При разрыве соединения игрок возвращается на ту же вершину с тем же инвентарем |
| Транзакционность | Изменения предметов и счетчиков должны быть атомарными (ACID) |
| Отсутствие потери данных | При краше сервера сохраненные сессии не теряются (дублирование на реплику) |
| Время восстановления после падения (RTO) | < 15 минут |

---

## 3. Безопасность (Security)

| Требование | Описание |
|------------|----------|
| Хеширование паролей | Использовать bcrypt (соль + хеш), не хранить пароли в открытом виде |
| Разделение ролей | Автор не может редактировать чужие квесты; игрок не может менять чужие оценки |
| Защита от SQL-инъекций | Использовать параметризованные запросы (prepared statements) |
| Защита от XSS | Экранирование пользовательского ввода (текст комментариев, названия квестов) |
| HTTPS | Все запросы только через HTTPS (шифрование трафика) |
| Аутентификация | JWT-токены с временем жизни (access token + refresh token) |

---

## 4. Удобство использования (Usability)

| Требование | Описание |
|------------|----------|
| Подсказки при создании условий | При выборе предмета/счетчика в условии — выпадающий список только из существующих |
| Drag-and-drop редактор графов | Для автора: визуальное создание вершин и связей перетаскиванием |
| Валидация перед публикацией | При нажатии "Опубликовать" — автоматическая проверка достижимости всех вершин |
| Адаптивный дизайн | Интерфейс должен корректно отображаться на экранах от 375px (мобильные) до 1920px |
| Поиск с автодополнением | При вводе названия книги — подсказки из существующих |

---

## 5. Масштабируемость (Scalability)

| Требование | Описание |
|------------|----------|
| Горизонтальное масштабирование БД | PostgreSQL с репликацией (master-slave) |
| Кэширование часто запрашиваемых данных | Redis для кэширования: список книг, топ-10 квестов |
| Stateless бэкенд | Серверная часть не хранит состояние пользователя (сессии в Redis/БД) |
| Балансировка нагрузки | Nginx или HAProxy между несколькими экземплярами бэкенда |

---

## 6. Доступность (Availability)

| Требование | Целевое значение |
|------------|------------------|
| Время работы (Uptime) | 99.5% (≤ 3.6 часов недоступности в месяц) |
| Плановое обслуживание | Раз в неделю, ночью (02:00 - 04:00) |
| Время восстановления после сбоя (RTO) | < 15 минут |
| Максимально допустимая потеря данных (RPO) | < 5 минут |

---

## 7. Технологический стек (Technology Stack)

| Компонент | Выбор | Обоснование |
|-----------|-------|-------------|
| **БД** | PostgreSQL 15+ | Поддержка JSON, рекурсивных CTE, надежность |
| **Бэкенд** | Python / FastAPI | Быстрая разработка, асинхронность, хорошая документация |
| **Фронтенд** | React + TypeScript | Компонентный подход, типизация |
| **Кэш** | Redis | Высокая скорость, поддержка TTL |
| **Деплой** | Docker + Docker Compose | Унификация окружения, простота развертывания |
| **CI/CD** | GitHub Actions | Автоматические тесты при пуше |
| **Мониторинг** | Prometheus + Grafana | Сбор метрик, визуализация |

---

## 8. Ограничения (Constraints)

| Ограничение | Значение |
|-------------|----------|
| Максимальный размер квеста | 1000 вершин, 5000 ребер |
| Максимальная длина текста вершины | 10000 символов |
| Максимальная длина комментария | 2000 символов |
| Максимальное количество предметов на квест | 500 |
| Максимальное значение счетчика | -2^31 … 2^31-1 (стандартный INTEGER в PostgreSQL) |
| Глубина рекурсии для валидации | 50 (защита от бесконечных циклов) |

---

## 9. Совместимость (Compatibility)

| Требование | Описание |
|------------|----------|
| Браузеры | Поддерживаются последние 2 версии Chrome, Firefox, Safari, Edge |
| ОС | Windows, macOS, Linux (через браузер) |
| Мобильные устройства | iOS 14+, Android 10+ (через браузер) |
