# Платформа для создания квестов

**Роль на проекте:** ИТ-аналитик (сбор требований, моделирование данных, документация)

 Проект демонстрирует навыки работы с требованиями, проектирования БД для сложной предметной области (ориентированные графы, предметы, счетчики) и написания технической документации.

---

## О проекте

Платформа для создания и прохождения квестов в формате "выбери своё приключение" (Choose Your Own Adventure).  
Авторы могут создавать нелинейные сюжеты с ветвлением, системой инвентаря, скрытыми счетчиками и таймерами.  
Игроки проходят квесты (анонимно или с сохранением прогресса), оценивают книги и оставляют комментарии.

**Ключевая сложность проекта:**  
Спроектировать БД для хранения ориентированного графа (вершины -> ребра -> условия -> эффекты) с поддержкой:
- Предметов и скрытых счетчиков в единой полиморфной таблице
- Условной видимости ребер
- Лимитов переходов
- Таймеров на вершинах
- Переноса инвентаря между связанными квестами

│   ├── 05_functional_requirements.md        # Функциональные требования (модули)
│   ├── glossary.md                          # Глоссарий (24 термина)
│   └── ТЗ_платформа_квестов.md              # Сводное ТЗ и ссылки на артефакты
├── business_processes/                      # Use Cases (варианты использования)
│   ├── UC-01_create_quest.md                # Создание квеста (автор)
│   ├── UC-02_play_quest.md                  # Прохождение квеста (игрок)

│   ├── schema.sql                           # DDL (CREATE TABLE, FOREIGN KEY, CHECK)
│   ├── data_dictionary.md                   # Описание всех таблиц и полей
