# Техническое задание на проектирование базы данных для социальной сети

## 1. Цель работы
Спроектировать реляционную базу данных для социальной сети. База данных должна обеспечивать хранение информации о пользователях, их взаимодействиях (дружба, сообщения), публикациях, лайках, комментариях и сообществах.

## 2. Хранимые данные

### 2.1. О пользователе
- Фамилия, имя, отчество
- Дата рождения
- Место проживания
- Место(а) работы и/или учебы
- Телефон, e-mail
- Интересы
- Статус
- Пароль
- Список сообществ (в которые входит)
- Список друзей (с кем дружит)

### 2.2. О сообщениях
- Текст сообщения
- Дата и время отправки
- Отправитель и получатель

### 2.3. О публикациях (постах)
- Текст публикации
- Дата публикации
- Список пользователей, поставивших лайк
- Комментарии (текст, автор, дата)

### 2.4. О сообществах
- Название
- Описание
- Организаторы (администраторы)
- Публикации (с указанием автора)
- Ссылки на медиафайлы
- Ссылки на другие сообщества

---

## 3. Запросы для реализации

### 3.1. Простые выборки и сортировка

1. Выбрать все данные о пользователях. Результат отсортировать по фамилии в порядке, обратном лексикографическому.
2. Выбрать годы рождения пользователей без повторений. Результат отсортировать в порядке возрастания года.
3. Выбрать фамилию и инициалы пользователей, рожденных в 1999, 1997, 2000, 1993, 1990 годах. Результат отсортировать по месяцу рождения от января до декабря.
4. Выбрать текст публикаций пользователя с id = 1, сделанных в текущем месяце.
5. Выбрать все данные о пользователях, рожденных с 1990 по 2000 год.
6. Выбрать все данные публикаций, текст которых содержит «ВГУ» или «ПММ 50 лет» Результат отсортировать по дате в порядке убывания и по id пользователя в убывающем порядке.
7. Выбрать данные публикаций, текст которых содержит символы: **«-»**, **«_»**, **«%»** или **«&»**, но не содержит **«ВГУ»**.
8. Выбрать данные сообществ, у которых не указано описание.
9. Выбрать фамилию, инициалы и дату рождения пользователя в одном столбце. Результат отсортировать по дате в убывающем порядке, по длине (фамилия + имя + отчество) в возрастающем порядке.
10. Выбрать id, фамилию, имя, отчество пользователя и в отдельном столбце указать название времени года, в котором пользователь празднует день рождения. Результат отсортировать: сначала пользователи 1990–2000 г.р., затем — после 2000 г., затем — все остальные.

### 3.2. Агрегатные функции и статистика

11. Выбрать название сообщества, идущего первым в списке сообществ, упорядоченных по названию в лексикографическом порядке.
12. Выбрать средний возраст пользователей.
13. Выбрать общее количество различных годов рождения пользователей.
14. Выбрать максимальный и минимальный возраст пользователей.
15. Выбрать год рождения самого молодого пользователя, рожденного весной.
16. Вывести фамилию, имя, отчество пользователя, даты и тексты всех публикаций. Результат отсортировать по ФИО в обратном лексикографическом порядке.
17. Вывести ФИО пользователя, даты/тексты публикаций, названия сообществ (подписки), тексты сообщений пользователя. Сортировка: по дате рождения (убыв.), по ФИО (обрат. лекс.), по дате публикации (возр.).
18. Выбрать ФИО всех друзей пользователя Иванова Ивана Ивановича.
19. Выбрать ФИО пользователей, которые отправляли сообщения друзьям и получали от них ответные сообщения.
20. Вывести имя пользователя, дату рождения и количество написанных им сообщений.

### 3.3. Условия на группы и подзапросы

21. Выбрать все данные пользователей, которые участвуют как минимум в двух сообществах.
22. Выбрать ФИО несовершеннолетних пользователей и количество их друзей. Сортировка по количеству друзей (убыв.).
23. Выбрать ФИО несовершеннолетних пользователей, у которых более пяти друзей. Сортировка по ФИО.
24. Выбрать все данные о пользователях — полных тезках (ФИО совпадают), рожденных в один день, но в разные годы.
25. Для каждого пользователя указать даты первой и последней публикаций. Включить пользователей старше 20 лет, входящих в сообщество *N*, с друзьями >3. Сортировка по ФИО.

### 3.4. Соединения (JOIN) и внешние соединения

26. Вывести данные всех пользователей и, если есть сообщения, — текст сообщений.
27. Вывести для каждого пользователя количество его сообщений.
28. Для каждого пользователя, имеющего друзей, вывести названия всех сообществ.
29. Для каждого пользователя вывести названия сообществ и в последнем столбце — +, если он является членом.
30. Для каждого пользователя выбрать количество людей, с которыми он обменивался сообщениями.

### 3.5. Сложные условия с EXISTS / ALL / ANY

31. Выбрать названия сообществ, в которых есть пользователи, сделавшие несколько публикаций в текущем месяце. Сортировка по названию (лекс.).
32. Выбрать пользователей, которые пишут сообщения только своим друзьям.
33. Выбрать все данные пользователей, у которых более *N* друзей.
34. Выбрать название сообщества и количество участников. Включить только сообщества с названием из ≥2 слов и участников > *N*.

### 3.6. Расширенные проекции и CASE

35. Выбрать всех пользователей и (если есть) название сообщества.
36. Выбрать всех пользователей, количество друзей и (если есть членство) количество сообществ.
37. Выбрать ФИО пользователя, ФИО его друзей, тексты сообщений, названия сообществ. Сортировка: по названию сообщества (лекс.), по фамилии (обр. лекс.), по имени (прям.), по отчеству (обр. лекс.). Включить всех пользователей (LEFT JOIN).

### 3.7. Кросс-таблицы и аналитика

38. Выбрать год и количество рожденных по временам года. Результат: 5 столбцов (год, зима, весна, лето, осень).
39. Выбрать ФИО пользователя, количество полученных сообщений и количество отправленных.
40. Для каждого совершеннолетнего — среднее количество постов в месяц. Только для членов сообществ.

### 3.8. Оконные функции и TOP-N

41. Для каждого пользователя вывести текст его последнего сообщения. Сортировка по id_пользователя.
42. Выбрать пользователей, которые еще не написали ни одного сообщения.
43. Выбрать имя пользователя, у которого больше всего публикаций в одном сообществе.
44. Выбрать названия трех сообществ, в которых меньше всего публикаций.
45. Выбрать имя пользователя с самым коротким паролем, который сделал публикации в сообществе с наибольшим количеством пользователей.

### 3.9. Множественные операции и пересечения

46. Выбрать пользователей, которые делали публикации ≥2 в сообществах с макс. числом публикаций И ≥2 в сообществах с мин. числом публикаций.
47. Выбрать тексты публикаций самого пожилого пользователя.
48. Выбрать пользователей, которые писали сообщения всем своим друзьям.
49. Вывести данные о сообществах, в которых нет пользователей.
50. Выбрать пользователей, которые еще не написали ни одного сообщения (повтор 42).

### 3.10. Экстремумы и анти-паттерны

51. Выбрать имя пользователя с наибольшим количеством сообщений.
52. Выбрать имя пользователя с самым длинным сообщением.
53. Выбрать ФИО пользователей, которые обменивались сообщениями, но не являются друзьями.
54. Выбрать ФИО пользователей, которые не отвечали на сообщения не-друзей.
55. Выбрать ФИО пользователя, у которого есть друзья, но нет обмена сообщениями с ними.

### 3.11. Полнотекстовый поиск и даты

56. Выбрать в одном столбце: названия сообществ, тексты сообщений, тексты постов, содержащие слова **«базы данных»**. Сортировка по длине строки.
57. Выбрать название сообщества с наибольшим количеством пользователей.
58. Выбрать все дни текущего года, в которые не было публикаций.
59. Сложный отчет по сообществу: организаторы, активные участники (комментарии/лайки), % активных, количество читающих и членов других сообществ.
60. **Проверить правило шести рукопожатий** (найти кратчайший путь между двумя любыми пользователями в графе друзей).
